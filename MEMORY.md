# Otto — session memory (for agents)

## Role

- **Otto** orchestrates global CLIs `opencode-ai`, `kimaki`, and merges opencode plugins into `~/.config/opencode/opencode.json`. It does **not** patch upstream packages.

## Version

- Distribution version is **`MANIFEST.version`** in `src/manifest.ts` (mirrors `package.json`); shown by `otto status` as “Otto version”.

## Config surface

- Otto writes user-global **`~/.config/opencode/opencode.json`** only (not `~/.kimaki/opencode-config.json`).
- **NEVER** add custom keys (like `otto`) to `opencode.json` — opencode has **strict config validation** and rejects unrecognized keys with `Error: Configuration is invalid — Unrecognized key: "otto"`. This **breaks** `kimaki send --wait` and any new opencode server startup.
- Otto's own config lives in **`~/.config/opencode/otto.json`** (separate file, read by `readOttoConfig()` / `writeOttoConfig()`).
- **`subagentThreads`** in `otto.json`: `enabled`, `askBeforeDelete`, `autoDeleteOnComplete`; policy text is generated by `buildSubagentThreadPolicy(ottoConfig)` and injected into every `agent.*.prompt` via `mergeAgentPrompts()`.
- Actual Discord thread create/archive is performed by **opencode/kimaki** when the agent follows the policy (e.g. `kimaki send`, `kimaki session archive`); Otto does not call Discord APIs itself.
- Skill file `~/.config/opencode/skills/otto-subagent-threads/SKILL.md` is created by `ensureSubagentThreadSkill()` for recursive subagent spawning.

## Upgrade flow

- **`planStableUpgrades()`** (`src/installer.ts`) is the single place that decides which packages differ from manifest `pinned`; used by **`otto upgrade stable`** in `src/cli.ts` to avoid duplicating filter logic.
- **`otto upgrade latest`** still upgrades every manifest package (unchanged behavior).

## Kimaki restart

- `kimaki restart` kills the current opencode session when run from inside that session; Otto skips restart when `process.env.KIMAKI` is set and prints a manual warning.

## Tests / verify

- `pnpm test` — `detect.test.ts` / `health.test.ts` are slow (~npm list -g).
- `pnpm build` — `tsc` then **`chmod +x dist/cli.js`** so global `npm install -g .` symlink to `dist/cli.js` is executable (avoids “Permission denied” on `otto`).
- **Global install** (`npm install -g .` from repo root after build) is often **user-side**; coding agents typically do not run it unless the user asks. Local check without global install: `node dist/cli.js status`.
